💡 Tips for token-efficient prompts

1
Drop the politeness. Models don't need "please", "kindly", or "thank you". Each one costs ~1 token without changing the response.
2
Prefer imperatives. "Summarize this." beats "Could you possibly summarize this for me?" — same output, ~7× fewer tokens.
3
Compress verbose phrases. "in order to" → "to", "due to the fact that" → "because", "at this point in time" → "now".
4
Use structured output formats. Asking for JSON/YAML often produces a tighter response and reduces output tokens (the expensive ones).
5
Cache long system prompts. If your provider supports prompt caching (Anthropic, OpenAI), put stable instructions first to slash repeated input cost.
6
Avoid redundant context. Don't repeat what's in the system prompt inside every user message — it's billed every turn.
